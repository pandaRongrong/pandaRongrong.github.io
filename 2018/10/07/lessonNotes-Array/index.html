极客时间 课程笔记之数组

发表于 2018-10-07 | 分类于 数据结构与算法分析 ， 极客时间


数组作为最简单的数据结构，其在计算机中是使用连续的一段内存进行存储数据，根据下标随机访问元素非常方便，但为了保持保证连续性，删除、插入操作需要大量搬移数据，时间复杂度较高

为什么很多编程语言中数组都从0开始编号

一. 课程主要内容

  1. 数组（Array）的定义：是一种线性表数据结构。它用一组连续的内存空间，来存储一组具有相同类型的数据。

    • 连续的内存空间和相同类型的数据，使得数组能够实现随机访问，但为了保证连续性，要在数组中删除、插入一个数据时需要大量搬移数据

    • 计算机是通过地址来访问内存中的数据。数组中某元素存储的内存地址公式为：

      a[i]_address=base_address+i*data_type_size

    • 数组适合查找操作，但即使是排序好的数组，用二分查找，时间复杂度也是O（logn）,数组根据下标随机访问时间复杂度为O(1)

  2. 数组的插入操作
    • 若数组中的数据有序，则将一个数据插入长度为n数组的第k个位置，最坏时间复杂度为O(n),平均情况时间复杂度为（1+2+…+n）/n=O(n)
    • 若数组中的数据无序,为了避免大规模的数据搬移，直接将第k位的数据搬移到数组元素的最后，把新的元素直接放入第k位置
  3. 数组的删除操作
    • 一般情况时间复杂度分析同插入操作，当我们并不一定非得追求数组中数据的连续性时，每次的删除操作并不是真正搬移数据，只是记录数据已经被删除，当数组没有更多空间存储数据时，我们再触发执行一次真正的删除操作（JVM标记清除垃圾回收算法的核心算法）
  4. 数组越界
    • 数组越界在C语言中没有规定数组访问越界时编译器应该如何处理，因为访问数组的本质就是访问一段连续内存，只要数组通过偏移计算得到的内存地址是可用的，那么程序就可能不会保任何错误
  5. 容器与数组
    • 容器的优势：可以将很多数组操作的细节封装起来，支持动态扩容，数组本身在定义的时候需要预先指定大小
    • 当数据大小事先知道，对数据操作非常简单，可以考虑用数组；当表示多维数组时，用数组会更加直观
    • 对于业务开发，直接使用容器，如果做非常底层的开发，数组优于容器

  1. 为什么大多数编程语言，数组要从0开始编写

    • “下标”最确切的定义为“偏移”，如果数组从1开始计数，计算数组元素a[k]的内存地址变为：

      a[k]_address=base_address+(k-1)*data_type_size

      我们会发现后一个公式多了一次减法运算，对于CPU来说，就是多了一次减法指令，对于二维数组的寻址公式，下标为0优势更明显

    • 历史原因

二. 课后思考题

  1. 课程思考题（课后同学留言讨论）

    1. 对JVM的标记清除垃圾回收算法的理解

      杰杰：大多数主流虚拟机采用可达性分析算法来判断对象是否存活，在标记阶段，会遍历所有 GC ROOTS，将所有 GC ROOTS 可达的对象标记为存活。只有当标记工作完成后，清理工作才会开始。
      不足：1.效率问题。标记和清理效率都不高，但是当知道只有少量垃圾产生时会很高效。2.空间问题。会产生不连续的内存空间碎片。

    2. 二位数组a[m][n]的内存寻址公式

      a[i][j]_address=base_address+ (i*n+j)*data_type_size

  2. 课后讨论问题

    1. 对于下述代码，当数组越界时会导致无限循环

      int main(int argc, char* argv[])
{
  int i = 0;
  int arr[3] = {0};
  for(; i<=3; i++){
    arr[i] = 0;
    printf("hello world\n");
  }
return 0;
}

      slvher: 函数体内的局部变量存在栈上，且是连续压栈。在Linux进程的内存布局中，栈区在高地址空间，从高向低增长。变量i和arr在相邻地址，且i比arr的地址大，所以arr越界正好访问到i。当然，前提是i和arr元素同类型，否则那段代码仍是未决行为.
